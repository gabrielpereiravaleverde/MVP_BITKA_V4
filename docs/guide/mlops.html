1.1.1 conf

Exemplos de arquivos nesta pasta:

  • catalog.yml: Descreve os datasets disponveis e suas fontes.
  • parameters.yml: Armazena parmetros de execuo, facilitando a flexibilidade e reusabilidade do cdigo.

1.2 Criao de um ambiente virtual via Conda

Para garantir a consistncia do ambiente de desenvolvimento, recomenda-se a criao de um ambiente virtual especfico para o projeto usando o Kedro como referncia.

1.2.1 Instalao do Conda

Se voc ainda no utiliza o Conda como gerenciador de ambientes virtuais, recomenda-se instal-lo seguindo as instrues em Conda Installation.

1.2.2 Criao de um Novo Ambiente Virtual com Conda
# Substitua `my_kedro_env` pelo nome que deseja dar ao ambiente virtual
conda create --name my_kedro_env python=3.7

# Ative o ambiente virtual
conda activate my_kedro_env

Ao executar esses comandos, voc ter criado um ambiente virtual isolado com o nome my_kedro_env e o Python 3.7 instalado. Certifique-se de ativar o ambiente virtual sempre que estiver trabalhando no projeto

1.2.2 Instalao das Dependncias do Projeto

Certifique-se de estar na raiz do projeto antes de executar os comandos a seguir.

# Instale as dependncias do projeto a partir do arquivo requirements.txt
pip install -r src/requirements.txt

# Inicialize o projeto Kedro
kedro install

Ao seguir esses passos, voc ter configurado um ambiente virtual com Conda e instalado as dependncias do projeto, conforme as melhores prticas do Kedro. Certifique-se de manter o arquivo requirements.txt atualizado medida que novas dependncias forem adicionadas ao projeto. Isso garantir consistncia entre ambientes de desenvolvimento.

2 Fluxo do Projeto

O fluxo do projeto abrange diversas etapas, desde o processamento de novos dados at a atualizao do Simulador MVP. Cada fase desempenha um papel crucial no desenvolvimento, calibrao e otimizao dos modelos preditivos. A seguir, detalhamos cada uma dessas etapas:

2.1 Processamento de Novos Dados

Nesta fase, os dados recm-coletados ou atualizados so processados e integrados ao projeto. Isso inclui a validao, limpeza e transformao dos dados para garantir sua qualidade e utilidade nos modelos preditivos.

2.2 Calibrao dos Modelos Preditivos

A etapa de calibrao dedicada ao ajuste fino dos modelos preditivos. Os algoritmos so treinados utilizando conjuntos de dados histricos, refinando seus parmetros para melhor se adaptarem s caractersticas especficas do sistema em questo.

2.3 Otimizao dos Hiperparmetros dos Modelos Preditivos

Nesta fase, so realizados experimentos para encontrar os melhores hiperparmetros dos modelos preditivos. Isso envolve a busca por combinaes ideais que maximizem o desempenho do modelo.

2.4 Verificao da Varivel que Captura Mudanas na Dinmica do Processo

A varivel que captura mudanas na dinmica do processo analisada para identificar padres e comportamentos significativos. Essa anlise contribui para a compreenso da dinmica do sistema e pode levar a ajustes nos modelos.

2.5 Anlise de Erro/Performance

A anlise de erro e desempenho dos modelos essencial para avaliar a qualidade das previses. Mtricas como preciso, recall, e erro mdio so examinadas para garantir que os modelos atendam aos critrios de desempenho estabelecidos.

2.6 Exportao do Modelo Preditivo

Os modelos preditivos calibrados e otimizados so exportados para serem utilizados em ambientes de produo. Esta etapa envolve a preparao dos modelos para implementao em escala real.

2.7 Atualizao do Simulador MVP

O Simulador MVP atualizado com os resultados dos modelos preditivos mais recentes. Isso garante que o simulador reflita as mudanas e melhorias identificadas durante o desenvolvimento do projeto.

Cada uma dessas etapas fundamental para o sucesso do projeto, contribuindo para a criao de modelos preditivos precisos e eficazes, bem como aprimorando continuamente o Simulador MVP.
